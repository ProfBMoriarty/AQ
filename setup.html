Setup

This page explains how to setup an HTML5 application for use with the AQ audio engine.

  • Installation

Installation

To use AQ in your HTML5 project, visit the download page and get the .zip file containing the current release of AQ.

Copy the aq1.0.js file into the same directory as your other scripts.

Add the following code to your HTML5 page:

<script src="aq.js" type="text/javascript"></script>

This script reference can be placed either in the <head> or <body> section of the page, but it must appear before any script(s) that call the AQ engine.

The AQ engine must be initialized before it can be used. Details are available in the API documentation for AQ.init().

Preparing audio files

You must provide a 16-bit .wav file (44.1 kHz or 22.05 kHz, mono or stereo) for every sound used by your application.

However, for better performance, it is strongly recommended that you also provide .ogg and .mp3 versions of the .wav file, stored in the same directory. Most browsers can handle at least one of these compressed formats, which load much faster. Encode at whatever bitrate you feel is adequate. For stereo music, I recommend at least 192 kbps for .mp3 files, and 128 kbps for .ogg files. Lower rates may be adequate for voices and sound effects.

You may also want to provide an .opus file, as this will (hopefully) soon become the standard for all browsers.

You can specify which file format(s) are available and should be preferred by AQ when loading. Details are available in the API documentation for AQ.load().

Important security issues

Webkit-based browsers, such as Google Chrome and Apple Safari, are picky about security. By default, they will not load resources (including audio files) that (1) do not reside on an actual HTTP server, and (2) do not share the root filepath of any script that tries to access them.

This means that you have to install all of your scripts (including aq.js) and audio files on a live server, and in the same root filepath, before you can test or deploy an application using Chrome and/or Safari. (It's possible to reconfigure these browsers to avoid this necessity, but I don't recommend it.)

A basic application directory might look like this:

/audio/   (a folder for your audio files)
aq.js   (the AQ engine script)
main.js   (your main application script)

Mozilla Firefox is much more lenient about security. Resource files don't need to be on a server, or share the same root filepath. This makes Firefox much easier to use for development. Unfortunately, Firefox doesn't support the Web Audio API yet, although Mozilla has announced its intention to begin supporting it in the near future.

Internet Explorer

Ignore it. Or detect it, and urge the hapless user to upgrade to a modern browser, such as Chrome or Firefox.
